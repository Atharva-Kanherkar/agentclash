# pr-263 — Test Contract

## Functional Behavior
- Binary scoring must fail when any declared dimension is unavailable, because every dimension is an implicit gate.
- Hybrid scoring must fail when any explicitly gated dimension is unavailable, because a required gate with no score should not be treated as passing.
- Weighted scoring may still compute an overall score from available dimensions, but explicitly gated dimensions that are unavailable must cause `passed=false`.
- When multiple agents tie on `overall_score`, winner selection must fall back to the legacy chain: correctness first, then reliability.
- The scoring, repository, API, and release gate behavior introduced by PR #263 must remain backward compatible for legacy four-dimension scorecards.
- The PR branch must merge cleanly with `main` after the fixes.
